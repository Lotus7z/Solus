## 2. Цель
Создание игры с глубоким сюжетом.

## 3. Пользователь
Обычный пользователь ПК (студент).

## 4. Функциональные требования (FR)
- FR1: игра в изометрии (Solus)
- FR2: полноценный сюжет
- FR3: 2d графика (В работе )
- FR4: разнообразные NPC

## 5. Нефункциональные требования (NFR)
- Приложение запускается без ошибок
- плавная работа игры: “стабильный Fps”
- Данные будут храниться локально (позже: JSON-файл)

## 6. Критерии приемки (как проверить)
- запуск через ярлык 
- Можно сохранить прогресс
- игра не вылетает 

## 7. План модулей (игра)
- Core (модели и логика)
- App (игровое меню)
